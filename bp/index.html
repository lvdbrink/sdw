datasets need to be identified

features (things) need to be identified

representations (geometry, topology) should be identified and must be if they are managed elsewhere.

Identifiers should be URLs (http URIs) and must be globally unique.

in order for identifiers to be useful, people should be comfortable creating them themselves (bottom up, like hashtags) and this BP will give some pointers about how to create good URIs (should be dereferenceable/resolvable). There is no top down authority that you have to go to in order to create identifiers for spatial objects. So just make them up yourself if you need them and they don't exist.

There are many mechanisms to reconcile (i.e. map/link) objects from different datasets. You can use spatial correlation, but this is often not enough. There is also danger in doing so. Then there's the question of using sameAs or e.g. skos mapping relations like skos:related etc.
